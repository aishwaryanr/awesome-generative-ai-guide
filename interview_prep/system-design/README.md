# AI System Design Case Studies

Worked AI system design interviews: a realistic question, a full answer, then the follow-ups an interviewer actually asks. Each case is built on one 5-layer spine, grounded in Problem-First design, backed by real production data, and it ships with runnable, provider-agnostic code you can execute.

This is the system design round for AI Engineers and Forward-Deployed Engineers, and a deep reference for anyone designing production AI systems.

## Cases

- **[Customer support agent](customer-support-agent/README.md)**: an e-commerce support agent that answers from a help center, looks up orders and opens tickets through tools, and escalates to a human when unsure. Walks retrieval, tools, memory, evals and guardrails, production, and optimization end to end, with a runnable LangGraph implementation.

More cases coming.

## How to read these

Start with the spine at the top of any case, then the scope, then walk the layers. Each case is self-contained. You can also point your coding agent at a case and have it walk you through interactively, or reimplement it in the SDK you use (see the callout at the top of each case).
