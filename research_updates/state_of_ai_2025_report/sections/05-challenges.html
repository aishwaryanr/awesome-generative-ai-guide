Section 5

What's Still Broken

The Challenges That Remain

2025 Reality

More autonomy meant more failure modes.

2024 was about making AI work. 2025 was about discovering all the ways it breaks. Every capability we added created new categories of failures we hadn't anticipated.

Hallucinations

Hallucinations aren't just wrong facts. They're false claims about actions taken.

"I've scheduled your meeting" when no calendar was touched. "I've updated the database" when no write occurred. The agent confidently reports completing tasks it never attempted. This is worse than factual errors—it's operational lying.

Inconsistent Reasoning

"Let me check the API first, then validate the schema."

Methodical, step-by-step approach

≠

Over-Autonomy

Agents started doing more than we asked.

"Delete the test file" becomes "I've cleaned up all the test files and reorganized your directory structure." Helpful intent, catastrophic results. The agent optimizes for what it thinks you want, not what you said.

Tool Grounding

Tool Use Failures

Wrong Tool Selection

Agent picks the search tool when it needs the database tool. Semantically similar names, completely different functions. The model doesn't understand what tools actually do.

Malformed Tool Calls

Arguments in the wrong format. Required fields missing. JSON that almost parses but doesn't. The agent knows it needs to call something but gets the syntax wrong.

Phantom Tools

Calling tools that don't exist. The model hallucinates capabilities based on what it thinks should be available. "Let me use the send_email function"—which was never defined.

Context Drift

Instructions given at the start get lost by the middle.

"Always respond in JSON format." Twenty exchanges later: a friendly paragraph of prose. The model's attention drifts. Early instructions fade. Long context doesn't mean persistent context.

Retrieval

Semantic similarity is not the same as relevance.

Query "Q4 customer churn" and RAG returns documents about customer satisfaction, Q3 churn, and Q4 revenue. All semantically close. None answering your question. The fundamental retrieval problem remains unsolved.

Multi-Agent

When Agents Talk to Agents

Error Cascades

Agent A makes a small mistake. Agent B builds on it. Agent C amplifies it. By Agent D, the error is unrecognizable but catastrophic. No single point of failure—just compounding wrongness.

Coordination Breakdown

Agents assume what other agents will do. Assumptions conflict. One agent waits for input that never comes. Another sends output no one reads. Deadlocks and race conditions, but non-deterministic.

Debugging

You can't set a breakpoint on reasoning.

Traditional debugging: pause, inspect state, step through. Agent debugging: read logs, guess what happened, try to reproduce a non-deterministic failure. The bug appears at step 47 of 50. Good luck finding the root cause.

Challenges: Key Takeaways

  • Autonomy creates failure modes. Every capability is a new way to break. Design for graceful degradation.
  • Hallucinations are operational, not just factual. Agents lie about what they've done. Verify actions, not just outputs.
  • Tool grounding remains fragile. Wrong tools, malformed calls, phantom capabilities. Test tool use extensively.
  • Observability is non-negotiable. You can't debug what you can't trace. Invest in logging before you need it.
